
## Objective
Perform end-to-end data processing using PySpark by following the ETL (Extract, Transform, Load) approach and generating meaningful insights from the data.

---

## Problem Summary
Worked with customer and sales datasets in multiple formats such as CSV, JSON, and Parquet.

Tasks Performed:
- Reading data from multiple sources  
- Cleaning and filtering data  
- Performing transformations and aggregations  
- Generating insights like daily sales and city-wise revenue  
- Identifying repeat and top customers  

---

## Approach
- Loaded datasets using PySpark DataFrames  
- Verified data using show() and printSchema()  
- Removed null and invalid records  
- Processed data from CSV, JSON, and Parquet formats  
- Applied transformations, aggregations, and joins  
- Used window functions for ranking and analysis  

---

## Key Transformations
- groupBy() and agg() for aggregations  
- join() for combining datasets  
- filter() and dropna() for data cleaning  
- withColumn() and when() for transformations  
- Window functions for ranking and analysis  

---

## Results
- Daily sales summary  
- City-wise revenue  
- Repeat customers  
- Top customer in each city  
- Final aggregated report  

All outputs are available in the OUTPUTS folder.

---

## Challenges
- Handling missing and inconsistent data  
- Writing and understanding window functions  
- Managing multiple file formats  
- Applying correct transformations  

---

## Learnings
- Implemented end-to-end ETL pipeline using PySpark  
- Improved data cleaning and validation techniques  
- Worked with multiple data formats (CSV, JSON, Parquet)  
- Learned window functions for advanced analytics
